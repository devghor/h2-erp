# Module Spec: Payroll — Salary Disbursement

**Status:** Planned
**Module Key:** `payroll`
**Parent Spec:** [4.1-payroll.md](4.1-payroll.md)

---

## Overview

Salary Disbursement is the payroll run engine. Each month (or as a special batch), HR creates a **Salary Disbursement Batch** that pulls every active employee's salary profile, snapshots per-head amounts, aggregates totals, and moves through an approval pipeline until salaries are marked disbursed.

Out of scope for this iteration: attendance-based deductions, NBR tax slab calculation, payslip PDF generation, bank advice file export.

## Status Flow

```
generated → processed → sent_for_approval ⇄ revert_from_approval
                                          ↓
                                 sent_for_disbursement → disbursed
```

| Status | UI Label | Who Triggers |
|---|---|---|
| `generated` | Draft | System on batch creation |
| `processed` | Processed | HR after review |
| `sent_for_approval` | Pending Approval | HR |
| `revert_from_approval` | Reverted | Approver |
| `sent_for_disbursement` | Pending Disbursement | Finance/Approver |
| `disbursed` | Disbursed | Finance — sets disbursement_date, marks all employees paid |

## Calculation Logic

```
For each active employee with an active salary profile:

1. Load PayrollEmployeeSalaryProfile with items + salary heads
2. For each profile item:
   - If is_basic_linked → amount = basic_amount × (basic_ratio / 100)
   - Else → use stored amount from profile item
3. gross_amount     = SUM of items where head_category IN (gross, benefit)
4. deduction_amount = SUM of items where head_category IN (deduction, adjustment)
5. net_amount       = gross_amount − deduction_amount
6. Snapshot head_name + head_category into batch_employee_items (is_adjustment = false)
7. Batch totals     = SUM(basic, gross, deduction, net) across all employees
```

**Future hooks (out of scope):**
```
After step 3, before step 4:
  Attendance:   per_day_rate    = gross / working_days_in_month
                absent_deduction = per_day_rate × absent_days
  PF employee:  pf_employee     = pf_rate × basic_amount
  Income tax:   annual_taxable  = (gross − pf_employee) × 12
                apply NBR slabs → monthly_tax = annual_tax / 12
  Add absent_deduction, pf_employee, monthly_tax as deduction items (is_adjustment = false)
```

## User Stories

- As HR, I can create a monthly salary disbursement batch for a given month/year so all employees' net salaries are computed automatically from their active profiles.
- As HR, I can add one-time adjustments (bonus, ad-hoc deduction) per employee in a draft batch before processing.
- As HR, I can process a draft batch to lock salary amounts and send it for finance approval.
- As Finance, I can revert a batch from approval back to HR if corrections are needed.
- As Finance, I can mark a batch as sent for disbursement after verifying all amounts.
- As Finance, I can disburse the batch by recording the actual payment date, which marks all employee records as paid.
- As HR, I can download a payslip PDF for any employee for any disbursed batch *(future)*.

## UI Notes

### Index Page (`/payroll/salary-disbursement-batches`)

- DataTable columns: Name, Period (Month Year), Type badge, Status badge, Employees, Net Total, Disbursement Date, Actions
- "Create Batch" button → `BaseDialog` with fields: Name (auto-suggest e.g. "May 2026 Monthly Salary"), Year (select), Month (select), Type (select), Remark; employee selection toggle: all active employees (default) or specific list by department/user
- Row actions: View (navigate to show), Delete (generated status only)

### Show Page (`/payroll/salary-disbursement-batches/{id}`)

**Header**: Batch name, Period badge, Status badge, Type badge

**Summary cards (4)**: Total Basic | Total Gross | Total Deduction | Total Net + Employee Count

**Action buttons** — context-sensitive, only valid next-step shown:

| Current Status | Available Actions |
|---|---|
| `generated` | Process Batch |
| `processed` | Send for Approval |
| `sent_for_approval` | Revert + Send for Disbursement |
| `sent_for_disbursement` | Disburse (dialog with disbursement_date input) |
| `disbursed` | None — read-only |

**Employee table**:
- Columns: Employee Code, Name, Designation, Basic, Gross, Deduction, Net, Payment Mode, Payment Status badge
- Expandable row or modal per employee showing salary-head breakdown (item table: Head Name, Category, Amount)
- Adjustment entry per employee available in `generated` status only (adds a row with is_adjustment = true)

**Status badge colours**: generated=gray, processed=blue, sent_for_approval=amber, revert_from_approval=orange, sent_for_disbursement=purple, disbursed=green
